2.4 Well written Javascript

first JavaScript

Tech: I applied different functions, showInputDialog() prompts the user to input data for each column of a new row. If any field is left empty, it alerts the user and prevents adding the row. addRow() adds a new row to the table with the provided data. deleteRow() removes the row containing the clicked delete button from the table. Together, these functions enable the dynamic addition and deletion of rows in the Source Log table based on user input and interaction.

Notes: The main Script source code can be viewed here. The simple add and delete buttons used inline JavaScript onclick event.

second JavaScript

Tech: Here, I've utilized a different JavaScript syntax, neither linking to an external script nor using inline scripting that I used before in the first JavaScript. The script sets up variables for the form, question container, next button, and submit button, initializing the current question index to 0. It manages an array called questions, representing each form question as objects with properties like label, input ID, input name, and type. The showCurrentQuestion() function dynamically updates the question container based on the current question index, generating HTML for different input types such as text and textarea. An event listener for the next button checks for empty input fields, increments the question index, and displays the next question. If it's the last question, a thank you message appears along with the submit button. Another event listener handles form submission, collecting data, constructing an email body, generating a mailto link, and opening it in the default email client.

Notes: You can check the Script source code in the script tag of HTML file.

Additonal Notes: My website is built using React, so its primary language is JavaScript-based. However, it utilizes many framework-specific syntaxes. Check out the source code of various components if interested.
